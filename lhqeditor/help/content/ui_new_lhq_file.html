User Interface New Localization File

The New Localization File window user interface contains the following components:

  • Name text field
  • Primary language dropdown
  • Country specific checkbox
  • Layout settings
  • Code generator template settings


Figure: New Localization File window user interface components

Name text field

This value will be displayed in the top (root) node in the navigation tree.

Note: The name can contain alpha-numeric characters only, and cannot start with a number. The reason is that the generated classes must satisfy these properties.

The name is used to generate classes.

Primary language dropdown

Country specific checkbox

Note: You can change this value later.

Layout settings

This setting allows you to define whether resources should be organized in a tree hierarchy or in a flat structure. When tree hierarchy is selected, you can also define if resources are allowed to be located directly in the root category.

Default values: categories and resources selected, allowed resources under root category checked

Note: You can change the layout settings later. See Project Settings [user interface].

Code generator template

This setting displays the project type of the LHQ file. Depending on the project type, associated code generator name is displayed below.

By clicking on Code Generator Settings label, the Code Generator Settings window opens, where you can configure how the code will be generated.

Note: Once the file is created, it is no longer possible to change its project type and the associated code generator.
