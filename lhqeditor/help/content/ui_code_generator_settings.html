User Interface Code Generator Settings

The window contains information and parameters related to code generation.

Generators create files of different extensions. Each extension has its specific parameters. Each extension files can be generated to different folders.


Figure: Code Generator Settings window user interface components

C# Generator

  • Generate methods with parameters checkbox - When checked, resources with parameters will have generated methods with parameters (using the defined suffix, see below). If not checked, resources with parameters will have methods generated without parameters.

    • Note: You should not use resources with parameters when this parameter is unchecked. Otherwise, resource text will be displaying parameter identifiers. Example:

    Note: Once you use parametrized resources and parametrized methods, and you want to change this parameter to unchecked, you will have to change the usage of the resource methods manually. Without it, you would get compilation error. Example:

    Default value: Checked

  • Suffix for methods with parameters text field - If a resource has parameters, the method name for the generated resource will have the selected suffix. Example: Method without parameter: Title(), method with parameters: TitleWithParams(...).

    • Default value: "WithParams"

  • Return primary language text on missing foreign translation checkbox - If checked, when a resource translation is missing in some language, the primary language text will be returned. Otherwise, empty string will be returned for this language.

    • Default value: Unchecked

  • Output folder text field - Output folder, where the C# generated files will be stored.

    • Default value: "Resources"

  • Use expression body syntax checkbox - When checked, expression body syntax will be used, which reduces size of the generated code. Examples:

    Note: Requires C# language level 7.0 or higher.

    Default value: Unchecked

  • Enabled - When checked, C# files are generated automatically every time the project is saved.

    • Default value: Checked

    Note: After the value is changed to checked, the files are re-generated.

ResX Generator

  • File name for primary language includes language code checkbox - When checked, primary language resx file name will also contain the language code. When unchecked, the file name will not contain the language code.

    • Note: Other languages resx file names always contain their language code.

    Note: When you change this setting, and you already have existing generated resx files, you should delete the file which used its previous setting. Example:

    Default value: Checked

  • Output folder text field - Output folder, where the resx generated files will be stored.

    • Default value: "Resources"

  • Enabled - When checked, resx files are generated automatically every time the project is saved.

    • Default value: Checked

    Note: After the value is changed to checked, the files are re-generated.

Reset to Default button - Sets default values for the parameters.
